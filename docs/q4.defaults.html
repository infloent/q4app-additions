q4.defaults.js, line 1

Child libraries have all the options and methods of this widget, but may add their own as well. See the child library's documentation for additional options or methods.
This library has the following child libraries:

  • q4.getTopOffset
  • q4.navLevelSlide
  • q4.scrollToSection
  • q4.sticky

Options

scrollSpeed string
Scroll speed for `scrollTo`.

Default value
1000
headerOffset string
Offset used with the `scrollTo` method to account for fixed headers.

Default value
0
mailingListSignupCls string
Any mailing list with this class will have their validation overwritten.

Default value
.module-subscribe--fancy
errorMessage string
Error message to display (i.e. mailing list signup / unsubscribe)

Default value
The following errors must be corrected
requiredText string
Text to display if an item is required for validation

Default value
is required
invalidText string
Text to display if an entry is invalid and failed validation

Default value
is invalid
captchaValidationText string
Text to display if captcha is required. `requiredText` and `invalidText` will often proceed this text (i.e Code is required)

Default value
Code
provideCodeText string
Text used if a code is required.

Default value
Please provide the code
errorTpl string
Custom template for email validation

Default value
'<p class="module_message module_message--error">{{errorMessage}}</p>' +
 '<ul>' +
     '{{#errors}}' +
         '<li>{{name}} {{message}}</li>' +
     '{{/errors}}' +
 '</ul>'
mailingListConfig object
Options for mainlist module.

Default value
{
 tpl: (
     '<div id="SubscriberConfirmation" class="module module-subscribe module-subscribe--fancy dark grid_col grid_col--3-of-6 grid_col--md-1-of-2">' +
         '<div class="module_container--outer">' +
             '<h2 class="module_title">Email Alerts</h2>' +
             '<div class="module_container--inner">' +
                 '<p class="module_message module_message--success"></p>' +
             '</div>' +
         '</div>' +
     '</div>'
 ),
 location: '.pane--footer'
 }
Parameters
tpl string
Template to overwrite mailing list signup confirmation html. See above default value.
location string
Location where the 'tpl' will be apended. See above default value.
superfish boolean
Enable superfish plugin

Default value
true

toggle
Creates a fully accessible expanding and collapsing accordion with the ability to switch between toggle and accordion functionality.

Parameters
element $container
the wrapping element for the toggle list
selector item
the class assigned to each designated toggling element
selector toggle
the class assigned to the element that will toggle the containing item
selector panel
the class assigned to the section that will be revealed if its containing item is toggled
boolean accordion
(optional) if true, the toggling section will take on accordion functionality
boolean allButton
(optional) if true, the toggling section will be accompanied by a "Hide All / Show All" button
boolean openFirst
(optional) if true, the first item will be set to active with its panel revealed
Example
app.toggle($('.accordion'), '.accordion_item', '.accordion_toggle', '.accordion_panel', false, true);
