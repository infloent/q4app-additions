This extends the 'q4.defaults' library. All the options and methods from 'q4.scrollToSection' will be inherited by 'q4.defaults' and will be available in q4App.
All the default options and methods from 'q4.defaults' are also inherited by 'q4.scrollToSection' and may be use here in the implementation of it's own options and methods.

Examples 

<script src="/files/js/q4.getTopOffset.1.0.0.min.js"></script>
<script src="/files/js/q4.scrollToSection.1.0.0.min.js"></script>
<script>
var q4App = $.extend(true, q4Defaults, {
    options: {
        //these tow options are available here since 'q4.getTopOffset' extends 'q4.defaults'
        offsetTopElem: [$('.pane--navigation .pane_outer')],
        offsetTopBreakPoint: [{
            breakPoint: 1023,
            offsetTopElem: []
        }]
    },
    init: function() {
        var app = this,
            q4Options: this.options;
        app.setHeaderOffset();
        //...
        //default q4 inits found already in main scripts here in between
        //...
        app.scrollToSection({
           menu: '.nav--main',
           level: '.level2',
           sectionClass: '.js__section--',
           hrefId: '#section=',
           scrollOnLoad: true,
           updateHrefHash: true,
           addActiveClass: true,
           closeMobileNav: true,
           offsetTopElem: q4Options.offsetTopElem,
           offsetTopBreakPoint: q4Options.offsetTopBreakPoint
        });
    }
});
q4App.init();
</script>

Methods

scrollToSection
Use this method to add scroll to section functionality to a navigation.

Parameters
options object
Object of options used to initialize 'scrollToSection' method
scrollToSection options:
menu string
Navigation element class.
If there are more navigation that use the same page sections for scroll and all the other options will be the same they can be intialize all at oance.

Default value
null
Example
".nav--main"
Example
".nav--main, .nav--secondary, .nav--mobile"
level string
Level class at which the scroll links will be displayed.
This is used to limit the script to check only the link from the selected link's child level.

Default value
null
sectionFocusElem string
Element common in all the section that will get focused after the scroll to active section finishes. This is required for accesibility.

This should be one of the top most element in the section, no other focusable elemenet should exist above 'sectionFocusElem' in the section.

Default value
.module_title
sectionClass string
Page section should have BEM like css class from a module(general) part and a modifier(unique).
These classes can start with a 'js_' prefix to point out that they will be used by a JavaScript. 
Example: 
js_scroll-section--intro
js_scroll-section--events
js_scroll-section--news
*Particular case:
If a section has it's own scroll navigation with subsections this option should be unique for each subsections group and so on.
And 'scrollToSection' should be initialized for each of these sections navigation. 
Example:
js_scroll-sectionintro--about
js_scroll-sectionintro--why-invest
js_scroll-sectionintro--legal

This option should be set with the module(general) part of the css class and include the modifier delimiter hence the two hyphens.

Default value
null
Example
"js_scroll-section--"
Example
"js_scroll-sectionintro--"
hrefId string
Each navigation link that will scroll to a section needs to have an href id at the end formed from a general part and a unique one. 
Example:
/events/default.aspx#section=intro
/events/default.aspx#section=events
/events/default.aspx#section=news
*Particular case:
If a section has a it's own scroll navigation with subsections this option should be unique for each subsections group and so on.
And 'scrollToSection' should be initialized for each of these sections navigation. 
Example:
/home/default.aspx#sectionintro=about
/home/default.aspx#sectionintro=why-invest
/home/default.aspx#sectionintro=legal


Default value
null
Example
"#section="
Example
"#sectionintro="
scrollSpeed number
Set scroll speed. By default it inherits the default 'scrollSpeed' from 'q4.defaults'.

Either change the default 'scrollSpeed' from 'q4.defaults' to affect all scroll animation using this setting or set this on initialization to a different number.

Default value
q4Options.scrollSpeed //q4Options points to general options from q4.defaults
scrollOnLoad Boolean
Activates scroll to section on page load when a link has a hrefId pointing to a section.

*Particular case:
If there are more initializations of 'scrollToSection' for the same sections with this option set to 'true' only the first initalization will actualy run this functionality to prevent conflicts.

Default value
false
updateHrefHash Boolean
Adds the complete 'hredId' to the window href when scroll links are clicked and dynamically on scroll for the current section in view port that's most near the top.

This is useful to share links that point to a section.

*Particular case:
If there are more initializations of 'scrollToSection' for the same sections with this option set to 'true' only the first initalization will actualy run this functionality on scroll to prevent conflicts.

Default value
false
addActiveClass Boolean
With this set to 'true' a 'js--active' class will be added on the currently clicked scroll link, also the class will pe dinamically added on scroll for the section that it's top most closer to the top of the window.

addScrollEvent Boolean
* not implemented

scrollSectionOffset Number
A way to add a number offset at which the scroll will stop from the top.
This can be either positive or negative number.

offsetTopElem array.<elements>
Array of elements that will be used to set scroll offset with 'getTopOffset' option.

An option with the same name it's also available in the options from 'q4.getTopOffset.js'.
*If the value of 'offsetTopElem' for the current initialization matches the one in the general options then it should point to the general option. See example at top of the page.

Default value
[] //empty array for no offset top
Example
offsetTopElem: [$('.pane--header'), $('.pane--banner')] // 'getTopOffset' will return the sum of these two elements heights.
offsetTopBreakPoint array.<object>
An array of objects containing two properties 'breakPoint' and 'offsetTopElem' the will be used to set set scroll offset with 'getTopOffset' option, on one or more given breakpoint resolutions.

An option with the same name it's also available in the options from 'q4.getTopOffset.js'.
*If the value of 'offsetTopBreakPoint' for the current initialization matches the one in the general options then it should point to the general option. See example at top of the page.

Default value
[] //empty array. In this case will calculte the offset only based on 'offsetTopElem'
Parameters
breakPoint number
offsetTopElem array.<elements>
Example
offsetTopBreakPoint: [{
    breakPoint: 1024,
    offsetTopElem: [$('.pane--header')] // 'getTopOffset' will return this element's height
  },{
    breakPoint: 768,
    offsetTopElem: [] //empty array for no offset top at resolutions below or equal tp 768px
}]
getTopOffset function
This option points to the method with the same name from 'q4.getTopOffset.js' and inheriths it's fucntionality. Click here for more details

This has no parameters set on init so it will use the local options 'offsetTopElem' and 'offsetTopBreakPoint'.

*Particular case:
If for some reason the default implementation of this method it's not enought, it can be completely rewriten on initialization.

Parameters
_offsetElem array
If not set this methotd will use 'this.offsetTopElem'
_offsetBreakPoint array.<object>
If not set this methotd will use 'this.offsetTopBreakPoint'
Returns
number
addActiveClassScrollTolerance number
Adds the ability to trigger the 'js--active' class on scroll sooner or later that it would by default.
This cand be either positive or negative number.

Default value
0
closePopup Boolean
Activates the functionality from 'onClosePopup' that by default will close the mobile navigation when a scroll link it's clicked.

By default the mobile nav it's opened by adding a 'js--mobile' class to the '.layout' element.

For other pop-ups to close with this default functionality, build them in the same way by adding a class to the '.layout' element.
e.g. a search pop-up should be opend with a 'js--search' class added to the '.layout' element.

Default value
false
popupShowingClass String
Class that will be removed with 'onClosePopup' method from the '.layout' element.
You can pass more classes separated by a space to this option.

Default value
js--mobile
Example
"js--mobile js--search js--custompopup" 
if one or more of these classes are found on the '.layout' element they will be removed when a scroll link it is clicked
onClosePopup function
When 'closePopup' it's 'true' removes the classes passed to 'popupShowingClass' option.

*Particular case:
If for some reason a custom popup was not implemented in the default way an can not be close with the defualt 'onClosePopup' functionality, more functionality cand be added on initialization by rewriting this method.
