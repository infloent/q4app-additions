This extends the 'q4.defaults' library. All the options and methods from 'q4.navLevelSlide' will be inherited by 'q4.defaults' and will be available in q4App.
All the default options and methods from 'q4.defaults' are also inherited by 'q4.navLevelSlide' and may be use here in the implementation of it's own options and methods.

Methods

navLevelSlide
Use this method to add stiky functionality to an element.

Parameters
options object
Object of options used to initialize 'navLevelSlide' method
navLevelSlide options:
$nav element
Navigation element

Default value
null
Example
$('.nav--mobile')
startLevel String
* not implemented
This allows to se set the level class from which the functionality will be added.

*This solves the problem when the 'home' page is set to show in nav, or when the '.level1' items are corporate link and the navigation it's styles to only show Investor relations items which are .level2 elements.

Default value
.level1
Example
'.level2'
slideStyle string
Defines a css class modifier from the default slide classes that can be uset to create custom slide animation using css. 
By default the following css classes will be used:
js--nav_slide-in
js--nav_slide-out
There are three slide style animation available in 'q4.navLevelSlide.css'
One with the default css classes.
And two more available by setting 'slideStyle' to "over" or "stack". See examples below.

Default value
null
Example
"over"
this will modify the default css slide classes to 'js--nav_slide-in--over' and 'js--nav_slide-out--over'
Example
"stack" 
this will modify the default css slide classes to 'js--nav_slide-in--stack' and 'js--nav_slide-out--stack'
nextArrow Boolean
Adds the element defined in 'nextArrowTpl' after each anchor that has children in the '$nav'.

If true the slide next functionality will be triggered by the element that has the class from 'nextArrowCls' option.
If false the silde next functionality will be triggered by each anchor that has children in the 'startLevel'.

Default value
true
nextArrowCls string
Class that will be use to trigger the slide next functionality on 'click' if 'nextArrow' it's true.

This class is available to be use in 'nextArrowTpl' as '{{nextArrowTpl}}'.

Default value
.nav_next-level
nextArrowIconCls String
This class is used to add a arrow icon to 'nextArrow' button
If another arrow icon is needed to be used, change this class without the need to change 'nextArrowTpl' default template.

Default value
.q4-icon_chevron-right
nextArrowTpl string
If the default template need to be changed a button element with the 'nextArrowCls' class should always be used.

A button element is required for accesibility because it's a focusable element and also emulates 'click' event handler by default on 'enter' keypress.
The 'nextArrowCls' class is required to be set on the 'button' because this class trigger the functionality.

The default template it's buid to be accesible, when creating a custom template please follow the accesiblity requirements.

Default value
'<button class="{{nextArrowCls}}" type="button">' +
     '<i class="{{nextArrowIconCls}}"></i>' +
     '<span class="sr-only">Open submenu ({{sectionTitle}})</span>' +
 '</button>'
