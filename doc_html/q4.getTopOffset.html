q4.getTopOffset

Adds functionality to set 'headerOffset' based on multiple elements and resolution breakpoins.

This extends the 'q4.defaults' library. All the options and methods from 'q4.getTopOffset' will be inherited by 'q4.defaults' and will be available in q4App.
All the default options and methods from 'q4.defaults' are also inherited by 'q4.getTopOffset' and may be use here in the implementation of it's own options and methods.

Examples 

<script src="/files/js/q4.getTopOffset.1.0.0.min.js"></script>
<script>
var q4App = $.extend(q4Defaults, {
    _options: $.extend(q4Defaults.options, {
        //these tow options below are available here since 'q4.getTopOffset' extends 'q4.defaults'
        
        // this array sould be set to all element that are fixed at the top to calculate the top scroll offset based on this elements.
        
        offsetTopElem: [$('.pane--header .pane_outer'), $('.pane--navigation .pane_outer')],
        
        // Because one of the offset element will be used as a mobile navigation on resolutions equal or less than 1023px this element will be hidden and not a fixed element anymore; and
        // Because on resolutions equal or less than 768px the remaining fixed element will not be fixed anymore, 'offsetTopBreakPoint' sould be set like in the example below:
        
        offsetTopBreakPoint: [{
            breakPoint: 1023,
            offsetTopElem: [$('.pane--header .pane_outer']
        },{
            breakPoint: 768,
            offsetTopElem: []
        }]
    }),
    init: function() {
        var app = this,
            q4Options: this.options;
            
        
        app.setHeaderOffset();
        //...
        //default q4 inits found already in main scripts here below
        //...
    }
});
q4App.init();
</script>

Options

offsetTopElem array.<elements>
Array of elements that will be used to set 'headerOffset' with 'setHeaderOffset' method.

Default value
[] //empty array for no offset top
Example
offsetTopElem: [$('.pane--header'), $('.pane--banner')] // 'getTopOffset' will return the sum of these two elements heights.
offsetTopBreakPoint array.<object>
An array of objects containing two properties 'breakPoint' and 'offsetTopElem' that will be used to set 'headerOffset' with 'setHeaderOffset' method one ore more given breakpoint resolutions.


Default value
[] //empty array. In this case will calculte the offset only based on 'offsetTopElem'
Parameters
breakPoint number
offsetTopElem array.<elements>
Example
offsetTopBreakPoint: [{
    breakPoint: 1024,
    offsetTopElem: [$('.pane--header')] // 'getTopOffset' will return this element's height
  },{
    breakPoint: 768,
    offsetTopElem: [] //empty array for no offset top at resolutions below or equal tp 768px
}]

Methods

getTopOffset
Method that returns the top offset number based on 'offsetTopElem' array and 'offsetTopBreakPoint' array of objects.

This method also checks if the elements passed to the 'offsetTopElem' array exists before calculating the offset.
In this way 'offsetTopElem', 'offsetTopBreakPoint', 'setHeaderOffset' can be initialized globally and will not throw an error if a sticky navigation it's not a global element and only shows on some pages.

Parameters
_offsetElem array
If not set this methotd will use 'this.offsetTopElem'
_offsetBreakPoint array.<object>
If not set this methotd will use 'this.offsetTopBreakPoint'
Returns
number
setHeaderOffset
Method that sets 'headerOffset' by calling on resize the 'getTopOffset' method.

This is useful for scripts that are using the 'headerOffset' option like the 'scrollTo' method from 'q4.defaults.js' that is used on other methods from 'q4.defaults.js'.
